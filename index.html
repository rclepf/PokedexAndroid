🧠 1. Arquitetura de Software e a Morte do Código Espaguete

No ecossistema Android clássico, a falta de padronização induzia os desenvolvedores a inflar as Activities com lógicas de rede, persistência e tratamento visual. O Android Moderno resolve isso utilizando separação estrita de escopo.

O Ecossistema MVVM

  • Model (Camada de Dados): Encapsula as regras de negócio, conexões de rede locais ou remotas e banco de dados. Totalmente agnóstico à interface visual.
  • ViewModel (O Detentor do Estado): Atua como o cérebro reativo da tela. Solicita informações ao Model, manipula e expõe dados através de UI States. **Importante:** Ele possui um ciclo de vida resiliente e não é destruído quando o usuário rotaciona a tela do celular.
  • View (A Camada Visual): Composta pelas funções @Composable do Jetpack Compose. É uma camada puramente passiva que apenas renderiza o estado atual do ViewModel.

O Mecanismo UDF (Unidirectional Data Flow)

Para mitigar estados inconsistentes ou falhas de sincronismo visual, implementamos o Fluxo Unidirecional de Dados:

1. O Estado Desce (State flows down): O ViewModel atualiza e despacha a estrutura de dados para as composables.
2. O Evento Sobe (Events flow up): A View capta as ações do usuário (cliques, digitação) e notifica o ViewModel por callbacks, sem alterar nenhuma variável por conta própria.

🌐 2. Consumo de APIs REST e Assincronismo de Alta Performance

Para interagir com servidores externos, realizamos chamadas HTTP para APIs REST. A resposta trafega na rede codificada em formato JSON (texto estruturado), que convertemos para objetos manipuláveis do Kotlin.

A Linha de Execução Crítica: Main Thread

O sistema operacional Android aloca a chamada Main Thread (ou UI Thread) unicamente para renderizar os frames da interface gráfica (60hz a 120hz). Qualquer requisição de rede ou processamento de arquivos bloqueante disparado nessa linha causará o congelamento total do app, resultando no erro fatal ANR (App Not Responding).

A Resolução com Kotlin Coroutines

Utilizamos as Coroutines e funções marcadas com o modificador suspend. Elas efetuam o desvio seguro da carga de processamento de rede para threads de segundo plano (Dispatchers.IO) de forma assíncrona, liberando a linha principal para manter as animações e interações do usuário perfeitamente fluidas.

package com.seunome.pokedex.model

// Mapeia o objeto raiz do JSON que contém a lista "results"
data class PokemonResponse(
    val results: List<PokemonItem>
)

data class PokemonItem(
    val name: String,
    val url: String
) {
    // Propriedade customizada para extrair o ID da URL e mapear a arte oficial
    val imageUrl: String
        get() {
            val id = url.split("/").dropLast(1).last()
            return "https://raw.githubusercontent.com/PokeAPI/sprites/master/sprites/pokemon/other/official-artwork/$id.png"
        }
}

Passo 4.2: Camada de Rede (Endpoints e Cliente HTTP)

Local do Arquivo: pacote network  |  Nome: PokeApiService.kt

network/PokeApiService.kt
package com.seunome.pokedex.network

import com.seunome.pokedex.model.PokemonResponse
import retrofit2.Retrofit
import retrofit2.converter.gson.GsonConverterFactory
import retrofit2.http.GET
import retrofit2.http.Query

interface PokeApiService {
    @GET("pokemon")
    suspend fun fetchPokemonList(
        @Query("limit") limit: Int = 151
    ): PokemonResponse
}

// Singleton de inicialização segura do Retrofit
object RetrofitClient {
    private const val BASE_URL = "https://pokeapi.co/api/v2/"

    val apiService: PokeApiService by lazy {
        Retrofit.Builder()
            .baseUrl(BASE_URL)
            .addConverterFactory(GsonConverterFactory.create()) // Conversor GSON integrado
            .build()
            .create(PokeApiService::class.java)
    }
}

Passo 4.3: O Cérebro da Tela (Gerenciamento de Estados de Rede)

Local do Arquivo: pacote viewmodel  |  Nome: PokedexViewModel.kt

viewmodel/PokedexViewModel.kt
package com.seunome.pokedex.viewmodel

import androidx.compose.runtime.getValue
import androidx.compose.runtime.mutableStateOf
import androidx.compose.runtime.setValue
import androidx.lifecycle.ViewModel
import androidx.lifecycle.viewModelScope
import com.seunome.pokedex.model.PokemonItem
import com.seunome.pokedex.network.RetrofitClient
import kotlinx.coroutines.launch

// Máquina de estados explícita mapeada para a nossa View
sealed interface PokedexUiState {
    object Loading : PokedexUiState
    data class Success(val list: List<PokemonItem>) : PokedexUiState
    data class Error(val message: String) : PokedexUiState
}

class PokedexViewModel : ViewModel() {

    // Estado observável encapsulado via State Hoisting interno
    var uiState: PokedexUiState by mutableStateOf(PokedexUiState.Loading)
        private set

    init {
        loadPokemons()
    }

    fun loadPokemons() {
        // Dispara uma Coroutine assíncrona no escopo seguro do ViewModel
        viewModelScope.launch {
            uiState = PokedexUiState.Loading
            try {
                val response = RetrofitClient.apiService.fetchPokemonList()
                uiState = PokedexUiState.Success(response.results)
            } catch (e: Exception) {
                uiState = PokedexUiState.Error(e.message ?: "Erro inesperado de conexão.")
            }
        }
    }
}

Passo 4.4: A Interface Gráfica Reativa

Local do Arquivo: pacote ui  |  Nome: PokedexScreen.kt

ui/PokedexScreen.kt
package com.seunome.pokedex.ui

import androidx.compose.foundation.layout.*
import androidx.compose.foundation.lazy.grid.GridCells
import androidx.compose.foundation.lazy.grid.LazyVerticalGrid
import androidx.compose.foundation.lazy.grid.items
import androidx.compose.material3.*
import androidx.compose.runtime.Composable
import androidx.compose.ui.Alignment
import androidx.compose.ui.Modifier
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.unit.dp
import androidx.lifecycle.viewmodel.compose.viewModel
import coil.compose.AsyncImage
import com.seunome.pokedex.model.PokemonItem
import com.seunome.pokedex.viewmodel.PokedexUiState
import com.seunome.pokedex.viewmodel.PokedexViewModel

@Composable
fun PokedexScreen(viewModel: PokedexViewModel = viewModel()) {
    Scaffold { paddingValues ->
        Box(
            modifier = Modifier.fillMaxSize().padding(paddingValues),
            contentAlignment = Alignment.Center
        ) {
            // Recomposições automáticas com base na alteração do Estado do Cérebro
            when (val state = viewModel.uiState) {
                is PokedexUiState.Loading -> CircularProgressIndicator(color = Color(0xFFE3350D))
                is PokedexUiState.Error -> Column(horizontalAlignment = Alignment.CenterHorizontally) {
                    Text("Erro: ${state.message}", color = Color.Red)
                    Button(onClick = { viewModel.loadPokemons() }) { Text("Tentar Novamente") }
                }
                is PokedexUiState.Success -> {
                    LazyVerticalGrid(columns = GridCells.Fixed(2), contentPadding = PaddingValues(10.dp)) {
                        items(state.list) { pokemon ->
                            Card(modifier = Modifier.padding(6.dp).fillMaxWidth()) {
                                Column(horizontalAlignment = Alignment.CenterHorizontally, modifier = Modifier.padding(12.dp)) {
                                    // Coil processando o link remoto e gerando cache local de forma automática
                                    AsyncImage(model = pokemon.imageUrl, contentDescription = pokemon.name, modifier = Modifier.size(90.dp))
                                    Text(pokemon.name, style = MaterialTheme.typography.titleMedium)
                                }
                            }
                        }
                    }
                }
            }
        }
    }
}
